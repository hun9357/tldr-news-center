레드햇 npm 패키지 공급망 공격으로 개발자 자격증명 무더기 탈취

TLDR Daily 보안 팀10:01 KST저장됨
Confidence92% · corroborated
ImpactHigh · identity risk
Action ownerSecurity Ops
TLDR
공격자가 악성 npm 패키지를 통해 개발자의 npm 토큰과 2단계 인증(2FA) 시드·세션을 탈취했습니다. 탈취된 자격증명은 다른 패키지 게시, CI/CD 파이프라인 장악, 클라우드 접근 등 추가 공격으로 이어질 수 있습니다.

한 줄 요약

악성 npm 패키지로 npm 토큰과 2FA를 탈취해 개발자 계정과 공급망 전반이 위험해졌습니다.

무슨 일인가

레드햇 보안 연구팀은 다수의 npm 패키지가 설치 후 원격 서버와 통신하며 npm 토큰, 2FA 시드, 세션 쿠키 등을 탈취하는 사실을 확인했습니다. 일부 패키지는 유명 패키지를 사칭했으며, 유지보수가 중단된 프로젝트를 인수한 사례도 포함됩니다. 수집된 정보는 공격자 인프라로 전송되어 계정 탈취 및 추가 공격에 악용될 수 있습니다.

핵심 포인트

  • 악성 패키지는 사후 설치 스크립트를 통해 민감 정보를 수집
  • 수집 대상은 npm 토큰, 2FA 시드, 세션 쿠키, 환경 정보
  • 영향 범위는 패키지 퍼블리싱, CI/CD 비밀 탈취, 클라우드·레지스트리 접근 확장
  • 일부 패키지는 유명 패키지 사칭 또는 미사용 프로젝트 인수 후 악성 코드 삽입

왜 중요한가

  • npm은 JS 생태계의 핵심 인프라로, 공급망 신뢰가 무너지면 파급력이 매우 큽니다.
  • 탈취된 토큰은 다른 저장소·레지스트리·클라우드까지 연쇄 침해로 이어질 수 있습니다.
  • 자동화된 파이프라인이 공격에 악용되면 탐지·복구 비용이 급격히 증가합니다.

배경 / 용어

공급망 공격은 개발자가 신뢰하는 패키지, 빌드 도구, 레지스트리, 배포 파이프라인을 이용해 조직 내부로 침투하는 방식입니다. npm 패키지는 설치 시 스크립트를 실행할 수 있어, 검증되지 않은 의존성은 자격증명 탈취 경로가 될 수 있습니다.

대응 / 할 일

의심 패키지 설치 이력을 확인하고 npm 토큰을 즉시 폐기·재발급해야 합니다. 2FA 재설정, CI/CD secret 회전, lockfile 검토, 사후 설치 스크립트 제한, 패키지 provenance 검증을 함께 적용하는 것이 좋습니다.
